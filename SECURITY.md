# 安全说明

## 适用范围
本项目只维护最新版的代码，旧版本不再处理安全问题。  

## 报告方式
如果你发现了漏洞，请不要直接开 Issue 或者发 PR。  
麻烦用 GitHub 自带的 **[安全报告功能](../../security/advisories/new)** 来提交，这样内容不会被公开

一般会在一周内回复并确认问题。修复之前，请不要在公共地方泄露漏洞细节。  

## 处理流程
1. 收到报告后先确认问题。  
2. 如果确实存在漏洞，会和你沟通细节。  
3. 修复完成后发布更新，必要时会在仓库里公开说明。
